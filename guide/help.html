# 其他问题

# docker重启但容器不重启

在默认情况下，当 docker daemon 停止时，正在运行的容器也会跟着停止。

在新版本的 docker 中（ > 1.12.6），可以启用 live-restore 选项，让容器保持运行状态。

进入 /etc/docker/daemon.json ，添加 "live-restore": true

{
    "live-restore": true,
}

然后进入 /usr/lib/systemd/system/docker.service，添加如下代码：

# kill only the docker process, not all processes in the cgroup
KillMode=process

最后重载一下 docker daemon

sudo systemctl daemon-reload

TIP

在 docker 1.12.6 版本后，KillMode 一般默认都是设置为 process。所以只需要添加 live-restore即可。但为了保险，仍需要检查一下 KillMode 的状态

具体可见官方详情 (opens new window)

# docker服务重启后容器启动失败

如果没有添加以上的代码，在docker服务重启后，容器启动可能会失败

使用该命令重启容器

# 该方法会重启所有的容器
docker ps -aq | xargs -I {} docker start {}

# 该方法可单独重启某个容器
docker ps 容器id | xargs -I {} docker start {}

# 修改部署网址

如果在开发过程中，想重新修改部署网址，则可以进入 Jenkinsfile 文件中进行修改

# Jenkinsfile

echo '==============Start Build=========='
cd /home/nginx/conf
sed -i "30a"\"}" nginx.conf
sed -i "30a"\"   }" nginx.conf
sed -i "30a"\"proxy_pass    http://172.17.16.17:\$port;" nginx.conf
sed -i "30a"\"location / {" nginx.conf
sed -i "30a"\"server_name  projectname.bluej.cn;" nginx.conf   # 在此处可修改部署后的域名
sed -i "30a"\"listen 80;" nginx.conf
sed -i "30a"\"server{" nginx.conf
sed -i "30a"\"  " nginx.conf
echo '==============End Build=========='

# 部署时打包失败

如果本地打包成功，但部署时却提示打包失败，一般是以下几种情况：

  • Node 版本过高，导致某些依赖包失效

    可以先在 Dockerfile 当中，查看 docker 指定的版本

    FROM node    		# docker 以最新的node版本创建容器

FROM node:10.x.x. 	# docker 以指定的node版本创建容器

    再看报错信息的来源，查找对应依赖包所支持的 node 版本，再自行选择升级 node 或是 降级 node

  • cnpm 导致某些依赖包安装不完整

    尝试将 dockerfile 中的以下代码进行修改，使用 npm 进行安装

    RUN npm install -g cnpm --registry=https://registry.npm.taobao.org
RUN cnpm install
# 删除以上两行代码
# 添加这段
RUN npm install

  • 调用某些依赖包时报错

    如果确定依赖包和 node 版本对应，并且切换 npm 安装仍然无法打包，可以尝试 rebuild 报错的依赖包来解决。

    FROM node AS builder
WORKDIR /app
COPY package.json /app/
RUN npm install --registry=https://registry.npm.taobao.org
COPY . /app
RUN npm rebuild node-sass     # 假如此处是由于 node-sass 依赖包调用错误，则添加该段代码
RUN npm run copy
