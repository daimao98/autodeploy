# 自定义配置

在这里，可以自定义目标服务器及其内容。除了列出来的配置之外，请不要改动其他文件。

# 非 egg 项目的部署

如果部署的后台不是 egg项目，则项目默认暴露的端口号可能与 Dockerfile 中的端口号不一致，所以需要进行端口号的修改。

FROM node AS builder
WORKDIR /app
COPY package.json /app/
RUN npm install -g cnpm --registry=https://registry.npm.taobao.org
RUN cnpm install
COPY . /app
RUN npm run build

FROM nginx
COPY --from=builder app/dist /usr/share/nginx/html/
COPY --from=builder app/nginx.conf /etc/nginx/conf.d/nginx.conf
EXPOSE 5566		# 比如某个项目默认后台端口为5566，此处可以改容器暴露的端口号

注意！！

如果更改了容器端口号，一定要在 Jenkinsfile 以及 nginx.conf 中，更改为对应的容器端口

# Jenkinsfile
docker run -d -p \$port:5566 --name bluej_projectname ${DOCKER_IMAGE_URL}

# nginx.conf
server 
{
    listen   5566;
}

# 静态项目的部署

由于静态资源不需要进行 build 打包操作，所以拉取进来后的目录结构会和默认打包的目录结构不一样，所以需要进行文件路径的修改。

FROM node AS builder
WORKDIR /app
RUN mkdir dist		# 创建 dist 目录
COPY . /app/dist	# 将项目扔进 dist 文件夹

FROM nginx
COPY --from=builder app/dist /usr/share/nginx/html/
COPY --from=builder app/dist/nginx.conf /etc/nginx/conf.d/nginx.conf	# 由于文件都扔进了dist，nginx也需要从dist拉取
EXPOSE 80

# 自定义目标服务器

// Jenkinsfile

// 声明服务器信息
def remote = [:]
remote.name = 'lyan-serve'
remote.allowAnyHosts = true
remote.host = '106.55.251.169'		// 此处可改目标服务器ip
remote.port = 22
remote.user = 'root'
remote.password = 'root'
