# 快速上手

# Step1. 从工蜂仓库拉取项目代码

  • 克隆 Vue 项目代码

    bluej initVue Vue项目名 要部署的域名

    拉取成功后，该目录结构下会生成一个名为 Vue项目名 的文件夹，项目文件夹内包含 初始的Vue模板，并且会出现三个新的文件：Dockerfile nginx.conf Jenkinsfile。如果需要将其他已开发的项目进行部署，则将初始的Vue模板替换为所需项目即可。

  • 拉取 Egg 项目代码

    bluej initEgg Egg项目名 要部署的域名

    拉取成功后，项目文件内出现两个新的文件：Dockerfile Jenkinsfile

    在部署后台项目的过程中，一定要注意 package.json 中的以下地方

    "scripts": {
	"start": "egg-scripts start",
},

    而不应该写成

    "scripts": {
	"start": "egg-scripts start --daemon",
},

    因为 --daemon 会导致项目在 docker 容器中以后台的方式运行，如果没有前台程序存在，docker 容器会自动退出，从而导致整个项目停止。

# Step2. 进行项目开发

bluej run Vue项目名			# 用于Vue项目开发
bluej runegg Egg项目名		# 用于Egg项目开发

该命令可进入项目的开发环境，就跟平时项目开发一个样子，之后即可进入项目的开发

# Step3. 向 git 仓库推送代码

首先在工蜂新建一个与项目同名的仓库，拿到仓库的地址，再通过自研命令进行代码推送

bluej toGit 项目名 仓库地址 备注

# bluej toGit vueProject https://git.code.tencent.com/daimao98/Jenkinsfile.git version1.0
# 备注选项是可选项，如果不填写，则默认值为 ”迁移“

# Step4. 配置Coding

进入项目并创建一个新的项目

进入后选择 DevOps 项目

进入 制品管理——制品仓库，新建一个制品仓库用来保存镜像

新建仓库时，制品仓库选择 Docker，权限范围选择 团队内

新建仓库后，进入到 持续集成 —— 构建计划，创建一个新的构建计划

在自定义构建过程中，必须按以下来配置

  • 构建计划名称必须和项目名相同

  • 代码源选择 工蜂

  • 代码仓库选择该项目的 git 仓库

  • 配置来源选择 使用代码库中的 Jenkinsfile

完成配置后，选择项目进入交互界面，点击立即触发

触发后，点击红色区域即可查看构建过程

部署完成！！
